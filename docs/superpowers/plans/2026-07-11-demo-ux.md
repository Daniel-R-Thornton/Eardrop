# Demo / Talk UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make eardrop presentable live on stage: type a message → hear it → watch it decode on the other machine, with visible transfer progress, signal health, clear failure guidance, and a fullscreen presentation mode.

**Architecture:** Builds on the modem-worker architecture plan (`2026-07-11-modem-worker-architecture.md`): all live data arrives as `ModemTelemetry` events at 20 Hz through `ModemController` and the `useTelemetry` hook — no new data paths are invented here. This plan adds: a text-message send mode, RX progress/ETA computed from `telemetry.progress`, a TX progress bar driven by playback timing, a large state banner, inline received-payload previews with a transfer history, a `?demo` fullscreen presentation layout, and actionable failure hints derived from telemetry (clipping, too quiet, sync flapping).

**Tech Stack:** React 18 (`useSyncExternalStore` stores already in place), TypeScript, vitest for logic, existing display components (`ConstellationPlot`, `SpectrumAnalyzer`, `ToneMeter`, `PipelineStrip`, `Card` in `src/ui/components/`).

**Prerequisites (hard):**
- Throughput plan implemented (`OFDM_TUNING`, 235 B frames, net-bitrate readout).
- Worker-architecture plan implemented through Task 5 (**`useTelemetry` hook, `ModemTelemetry.progress`, `ModemController`** — Tasks 2, 3, 5, 7 below consume them). If not, STOP and do that plan first.

- No new global state: UI prefs → existing `Store.ts`; live data → `useTelemetry`; transient view state → component `useState`.

```ts
/** Turns 20 Hz RxProgress snapshots into progress %, live B/s, and ETA. */
import type { RxProgress } from '../../workers/modemSchema';

export interface TransferView {
  active: boolean;
  pct: number;
  bytesPerSec: number;
  etaSec: number | null;
  fileName: string;
}

const IDLE: TransferView = { active: false, pct: 0, bytesPerSec: 0, etaSec: null, fileName: '' };

export class TransferTracker {
  private startMs: number | null = null;
  private startBytes = 0;
  private lastFile = '';

  update(p: RxProgress, nowMs: number): TransferView {
    const active = p.fileName !== '' && p.fileSize > 0 && p.bytesAssembled < p.fileSize;
    if (!active) {
      this.startMs = null;
      return { ...IDLE, fileName: p.fileName, pct: p.fileSize > 0 ? 100 : 0, active: false };
    }
    if (this.startMs === null || p.fileName !== this.lastFile) {
      this.startMs = nowMs;
      this.startBytes = p.bytesAssembled;
      this.lastFile = p.fileName;
    }
    const elapsedSec = (nowMs - this.startMs) / 1000;
    const pct = Math.min(100, Math.round((p.bytesAssembled / p.fileSize) * 100));
    const bytesPerSec = elapsedSec > 0.5 ? (p.bytesAssembled - this.startBytes) / elapsedSec : 0;
    const remaining = p.fileSize - p.bytesAssembled;
    const etaSec = bytesPerSec > 1 ? remaining / bytesPerSec : null;
    return { active: true, pct, bytesPerSec, etaSec, fileName: p.fileName };
  }
}
```

```ts
async function playWithProgress(samples: Float32Array, rate: number, deviceId?: string): Promise<void> {
  const durationSec = samples.length / rate;
  const t0 = performance.now();
  const timer = window.setInterval(() => {
    const elapsed = (performance.now() - t0) / 1000;
    const pct = Math.min(100, (elapsed / durationSec) * 100);
    setState({ txProgress: { pct, secondsLeft: Math.max(0, durationSec - elapsed) } });
  }, 100);
  try {
    await player.play(samples, rate, deviceId);
  } finally {
    clearInterval(timer);
    setState({ txProgress: null });
  }
}
```

```ts
import type { RxProgress } from '../../workers/modemSchema';

export function rxStateLabel(
  p: RxProgress | null,
  listening: boolean,
): { label: string; tone: 'idle' | 'active' | 'success' | 'error' } {
  if (!listening || !p) return { label: 'Idle — not listening', tone: 'idle' };
  switch (p.state) {
    case 0: return { label: 'Listening for signal…', tone: 'idle' };
    case 1: return { label: 'Preamble detected — syncing…', tone: 'active' };
    case 2:
      if (p.totalFrames > 0) {
        return {
          label: `Receiving "${p.fileName}" — frame ${p.framesReceived}/${p.totalFrames}`,
          tone: 'active',
        };
      }
      return { label: 'Signal locked — decoding…', tone: 'active' };
    case 3: return { label: `Received "${p.fileName}" ✓`, tone: 'success' };
    default: return { label: 'Decoder error — restart listening', tone: 'error' };
  }
}
```

```ts
function classifyPayload(name: string, bytes: Uint8Array): { kind: 'text' | 'image' | 'binary'; text?: string } {
  if (/\.(png|jpe?g|gif|webp)$/i.test(name)) return { kind: 'image' };
  // Heuristic: valid UTF-8 and mostly printable → text
  try {
    const text = new TextDecoder('utf-8', { fatal: true }).decode(bytes);
    const printable = [...text].filter((c) => c >= ' ' || c === '\n' || c === '\t').length;
    if (printable / Math.max(1, text.length) > 0.95) return { kind: 'text', text };
  } catch { /* not UTF-8 */ }
  return { kind: 'binary' };
}
```

```ts
/** Actionable mic-signal hints for the stage. Thresholds are empirical:
 *  peak > 0.95 clips the ADC; rms < 0.002 is below any decodable level. */
import type { ModemTelemetry } from '../../workers/modemSchema';

export function signalHealth(
  t: ModemTelemetry | null,
  listening: boolean,
): { level: 'ok' | 'warn'; hint: string } | null {
  if (!listening || !t) return null;
  if (t.peak > 0.95) {
    return { level: 'warn', hint: '⚠ Input clipping — lower mic gain or move devices apart' };
  }
  if (t.rms < 0.002) {
    return { level: 'warn', hint: '🔈 Input very quiet — raise speaker volume or mic gain, move closer' };
  }
  return null;
}
```

3. **Sound-reactive transmit animation** — pure cosmetics.

5. **QR code fallback slide** when audio fails on stage — presentation logistics, not code.

## What NOT to do

- Don't compute anything in components — every derived value above has a lib function and a test.
- Don't put `msgText`, demo-mode flags, or preview state in the global Store — they are component-local.
- Don't add telemetry fields casually — each one crosses the worker boundary 20×/s. Batch new needs into one schema change (see Appendix 1+2).
- Don't block the demo flow on `requestFullscreen` failures (it throws when not user-gesture-triggered) — always `.catch(() => {})`.
- Don't rename existing DOM events (`eardrop-send`, `eardrop-listen`, …) — `app.ts` and `MainApp.tsx` couple through them; this plan only ADDS `eardrop-send-text`.
